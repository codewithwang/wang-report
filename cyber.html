From the desk · Daily take
By Kai TannerCyber Intel Desk, Senior CorrespondentSun, Jul 19

Inc Ransomware's chaining of the two SonicWall SMA zero-days gets root on the appliance, which means the actor sits below the point where the appliance's own logging can see it. SonicWall's advisory frames the pair as "vulnerabilities allowing unauthorized access." Root-level code execution on a VPN gateway is not unauthorized access in the sense a compliance officer means when filling out a breach questionnaire. It is the ability to rewrite the thing that would have told you a breach happened.

The appliance in question is the same SMA product line whose earlier CVEs have shown up in Mandiant's and CISA's advisories as edge-device persistence points precisely because SMA boxes sit outside the telemetry most SOCs actually monitor: no EDR agent runs on the appliance itself, so root there is root nobody is watching. The control that would have changed this outcome is not patching velocity, which nobody in an active Inc campaign gets right in time anyway. It is treating SMA and equivalent VPN gateways as untrusted network segments with their own out-of-band log shipping, the same design principle HKMA's TM-G-1 already expects of internet-facing infrastructure carrying customer data. Firms that segmented the gateway logs before July 19, 2026 have an audit trail. The rest are relying on SonicWall's own box to tell them what happened inside it.

The piece reframes SonicWall's own advisory language as inadequate but never states what a correct advisory disclosure would have said instead, so the correction is asserted rather than demonstrated. It also treats TM-G-1 segmentation as a solved design principle without naming what changes for an SMA box that must still terminate remote-access sessions. -- WR
